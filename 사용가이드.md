## 프로젝트 소개

Modern Shop은 React와 Node.js로 만든 현대적인 풀스택 쇼핑몰입니다.

### 주요 특징
- ✨ 깔끔하고 모던한 디자인
- 📱 모바일 완벽 지원 (반응형)
- 🔐 안전한 JWT 인증
- 🛒 실시간 장바구니
- 📦 주문 관리 시스템
- 👨‍💼 강력한 관리자 패널

### 기술 스택
- **프론트엔드**: React 18, React Router, Context API
- **백엔드**: Node.js, Express, PostgreSQL
- **인증**: JWT (JSON Web Token)
- **보안**: bcrypt 비밀번호 암호화

## 설치 방법

### 필수 프로그램
1. **Node.js** (버전 16 이상)
   - [다운로드](https://nodejs.org/)
   
2. **PostgreSQL** (버전 12 이상)
   - [다운로드](https://www.postgresql.org/download/)

### 설치 단계

#### 1단계: 프로젝트 다운로드
```bash
# Git으로 클론하거나 ZIP 파일 다운로드
cd modern-shop
```

#### 2단계: 패키지 설치
```bash
npm run install-all
```
이 명령어 하나로 모든 필요한 패키지가 설치됩니다!

#### 3단계: 데이터베이스 생성
PostgreSQL을 실행하고:
```sql
CREATE DATABASE modern_shop;
```

#### 4단계: 환경 설정
`backend` 폴더에 `.env` 파일을 만들고:
```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=modern_shop
DB_USER=postgres
DB_PASSWORD=여기에_비밀번호_입력
JWT_SECRET=아무_문자열이나_입력
PORT=5000
NODE_ENV=development
```

#### 5단계: 데이터베이스 초기화
```bash
cd backend
npm run init-db
```

샘플 데이터가 자동으로 생성됩니다:
- 관리자 계정
- 테스트 사용자 계정
- 4개 카테고리
- 8개 샘플 상품

#### 6단계: 서버 실행
```bash
cd ..
npm run dev
```

브라우저에서 http://localhost:3000 접속!

## 사용자 가이드

### 회원가입하기

1. 상단 메뉴에서 "회원가입" 클릭
2. 정보 입력:
   - 이름
   - 이메일
   - 비밀번호 (6자 이상)
   - 비밀번호 확인
3. "회원가입" 버튼 클릭
4. 자동으로 로그인됩니다!

### 상품 구매하기

#### 1단계: 상품 찾기
- 홈페이지에서 "쇼핑 시작하기" 클릭
- 또는 상단 메뉴 "상품" 클릭
- 카테고리 필터나 검색 사용

#### 2단계: 상품 상세보기
- 원하는 상품 클릭
- 상품 정보 확인
- 수량 선택

#### 3단계: 장바구니에 담기
- "장바구니에 담기" 버튼 클릭
- 상단의 장바구니 아이콘에 숫자 표시

#### 4단계: 장바구니 확인
- 상단 메뉴 "장바구니" 클릭
- 수량 조절 가능
- 불필요한 상품 삭제 가능

#### 5단계: 주문하기
- "주문하기" 버튼 클릭
- 배송 정보 입력:
  - 수령인 이름
  - 연락처
  - 배송 주소
- "결제하기" 버튼 클릭

#### 6단계: 주문 완료
- 주문 완료 메시지 확인
- 주문 상세 페이지로 이동
- "주문내역"에서 언제든 확인 가능

### 주문 내역 확인하기

1. 상단 메뉴 "주문내역" 클릭
2. 모든 주문 목록 확인
3. 주문 클릭하여 상세 정보 보기:
   - 주문 상품
   - 배송 정보
   - 결제 금액
   - 주문 상태

## 관리자 가이드

### 관리자 로그인

기본 관리자 계정:
- 이메일: `admin@shop.com`
- 비밀번호: `admin123`

⚠️ **중요**: 첫 로그인 후 반드시 비밀번호를 변경하세요!

### 관리자 패널 접속

1. 관리자 계정으로 로그인
2. 상단 메뉴에 "관리자" 링크 표시
3. "관리자" 클릭하여 패널 접속

### 상품 관리하기

#### 새 상품 등록
1. 관리자 패널 > "상품 관리"
2. "상품 추가" 버튼 클릭
3. 정보 입력:
   - **상품명** (필수)
   - **가격** (필수)
   - **재고** (필수)
   - 상품 설명
   - 카테고리
   - 이미지 URL
   - 활성 상태 체크
4. "저장" 버튼 클릭

#### 상품 수정
1. 상품 목록에서 "수정" 버튼 클릭
2. 정보 변경
3. "저장" 버튼 클릭

#### 상품 삭제
1. 상품 목록에서 "삭제" 버튼 클릭
2. 확인 메시지에서 "확인" 클릭
3. ⚠️ 삭제된 상품은 복구할 수 없습니다!

### 카테고리 관리하기

#### 카테고리 추가
1. 관리자 패널 > "카테고리 관리"
2. 오른쪽 폼에 정보 입력:
   - **이름** (필수): 예) 의류
   - **슬러그** (필수): 예) clothing
   - 설명: 예) 패션 의류
3. "추가" 버튼 클릭

#### 카테고리 수정
1. 목록에서 "수정" 버튼 클릭
2. 오른쪽 폼에 정보가 자동 입력됨
3. 정보 수정
4. "수정" 버튼 클릭

#### 카테고리 삭제
1. 목록에서 "삭제" 버튼 클릭
2. 확인 메시지에서 "확인" 클릭
3. ⚠️ 해당 카테고리의 상품들은 카테고리가 없어집니다

### 주문 관리하기

#### 주문 목록 보기
1. 관리자 패널 > "주문 관리"
2. 모든 사용자의 주문 확인 가능
3. 주문번호 클릭하여 상세 정보 보기

#### 주문 상태 변경
1. 주문 목록에서 상태 드롭다운 클릭
2. 새 상태 선택:
   - **주문 접수**: 새로운 주문
   - **처리중**: 주문 처리 시작
   - **배송중**: 배송 시작
   - **배송 완료**: 배송 완료
   - **취소됨**: 주문 취소
3. 자동으로 저장됨

### 대시보드 확인하기

1. 관리자 패널 > "대시보드"
2. 주요 통계 확인:
   - 전체 상품 수
   - 전체 주문 수
   - 카테고리 수
   - 대기중인 주문 수

## 문제 해결

### 자주 묻는 질문 (FAQ)

#### Q1: 서버가 시작되지 않아요
**A**: 다음을 확인하세요:
1. PostgreSQL이 실행 중인가요?
2. `.env` 파일이 `backend` 폴더에 있나요?
3. `.env` 파일의 비밀번호가 정확한가요?
4. 데이터베이스 `modern_shop`이 생성되었나요?

#### Q2: "Module not found" 에러가 나요
**A**: 
```bash
npm run install-all
```
다시 실행해보세요.

#### Q3: 로그인이 안 돼요
**A**: 
- 이메일과 비밀번호를 정확히 입력했나요?
- 기본 계정으로 테스트해보세요:
  - user@shop.com / user123
  - admin@shop.com / admin123

#### Q4: 장바구니에 상품이 안 담겨요
**A**: 
- 로그인이 되어 있나요?
- 상품의 재고가 있나요?
- 브라우저 콘솔에 에러가 있나요? (F12)

#### Q5: 관리자 메뉴가 안 보여요
**A**: 
- 관리자 계정으로 로그인했나요?
- 일반 사용자 계정으로는 관리자 메뉴가 보이지 않습니다

#### Q6: 이미지가 안 보여요
**A**: 
- 이미지 URL이 올바른가요?
- URL이 https로 시작하나요?
- 추천: Unsplash 같은 무료 이미지 서비스 사용

### 에러 메시지별 해결법

#### "Error: connect ECONNREFUSED 127.0.0.1:5432"
**원인**: PostgreSQL이 실행되지 않음

**해결**:
```bash
# Windows
services.msc에서 PostgreSQL 서비스 시작

# macOS
brew services start postgresql

# Linux
sudo systemctl start postgresql
```

#### "Error: password authentication failed"
**원인**: 데이터베이스 비밀번호가 틀림

**해결**: `.env` 파일의 `DB_PASSWORD` 확인

#### "Port 3000 is already in use"
**원인**: 다른 프로그램이 포트 사용 중

**해결**:
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID [프로세스번호] /F

# macOS/Linux
lsof -ti:3000 | xargs kill
```

### 데이터 초기화

모든 데이터를 처음부터 다시 시작하려면:

```bash
cd backend
npm run init-db
```

⚠️ **주의**: 모든 데이터가 삭제되고 샘플 데이터로 대체됩니다!

## 팁과 권장사항

### 개발 팁

1. **코드 수정 후 자동 새로고침**
   - 프론트엔드: 자동으로 새로고침됨
   - 백엔드: nodemon이 자동으로 재시작

2. **API 테스트**
   - Postman 또는 Thunder Client 사용 권장
   - http://localhost:5000/api/health로 테스트

3. **데이터베이스 확인**
   - pgAdmin 또는 DBeaver 사용 권장
   - 테이블 구조 확인 가능

### 보안 권장사항

1. **프로덕션 배포 전**:
   - JWT_SECRET 변경 (복잡한 문자열)
   - 기본 비밀번호 모두 변경
   - HTTPS 사용
   - 환경 변수 보호

2. **정기적으로**:
   - 패키지 업데이트
   - 보안 패치 적용
   - 로그 확인

### 커스터마이징

1. **디자인 변경**:
   - `frontend/src/index.css`에서 색상 변경
   - 각 컴포넌트의 CSS 파일 수정

2. **새 기능 추가**:
   - 백엔드: `controllers`와 `routes` 추가
   - 프론트엔드: `pages`와 `components` 추가

3. **샘플 데이터 변경**:
   - `backend/scripts/initDb.js` 수정
